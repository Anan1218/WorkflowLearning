The opportunity in one line

What RLI wants: get submissions and policies in the door, read them, structure the data, decide which to work first, route to the right underwriter, and auto-adjudicate the simple ones. This is a named, well-mapped pattern in commercial underwriting: submission ingestion and triage, inside the broader submit-to-quote workflow.

Why build in-house

Generic platforms work "okay" then need heavy modification for surety and specialty lines. Owning the system fits RLI's unique book better than any catch-all.

Why now

Agentic tooling has lowered the build barrier dramatically. The calculus has shifted from "buy a vendor" toward "build our own, with the right partner."

Where Stello fits

Reference architecture · eight swappable layers

1.Intake. Multi-channel by design: monitored mailbox (email is dominant) plus SFTP, API, upload UI. Attachments burst to object storage and a queue.
2.Classification. Label each attachment (ACORD 125 vs loss run vs SOV vs financials vs WIP) using few-shot LLM prompting.
3.IDP / OCR. Read pixels/PDF into text, tables, key-value pairs, handwriting (e.g. AWS Textract). Clean digital PDFs can go straight to a multimodal LLM.
4.LLM extraction. Map read text into a typed JSON schema via tool-use / structured output, with per-field confidence.
5.Normalization / data plane (the moat). Collapse heterogeneous inputs into one canonical "golden schema" per line. This is the layer a generic vendor cannot tailor to surety for you.
6.Clearance / dedup. Entity resolution against the existing book (named insured, FEIN, address, broker) to decide new vs duplicate vs renewal.
7.Appetite / scoring. Encode RLI guidelines as rules + a model. Phased rollout: capture 5-7 key fields to filter at intake, expand to 10-20 with enrichment, then full risk profiling.
8.Routing, HITL, STP, write-back. Auto-decline clear misses, STP the clean simple risks, route the rest to a confidence-ranked queue, then write structured output back to legacy cores (API where possible, RPA / 3270 screen automation where not).

Surety & specialty · why in-house is justified

  • Three-party structure: the surety guarantees the obligee that the principal will perform. Two categories: contract and commercial. Principal, obligee, bond type, and bond amount are first-class entities that do not exist in a P&C schema.
  • The WIP schedule is the crown jewel. The Work-in-Progress schedule (per-project contract amounts, estimated costs, progress-to-date, under- and over-billings) is the surety equivalent of loss runs. Dynamic and project-level, not a static balance sheet. Often messy contractor Excel/PDF, a high-value extraction target.
  • Bonding capacity (single and aggregate) is computed from the WIP plus financials. Triage must calculate remaining capacity, not just classify.
  • Trucking / transportation needs a live FMCSA CSA/SMS data feed. 92% of commercial trucking underwriters use it in pricing, a different enrichment source than property or casualty.

Accuracy & compliance · table stakes for a public carrier

  • Extraction is non-deterministic. Never treat LLM output as exact. Wrap every extraction in schema validation, type/range checks, per-field confidence, and retries.
  • Confidence-thresholded human-in-the-loop. Auto-process only above a confidence bar (higher for surety financials than for a name field); route the rest to an underwriter.
  • Audit trails. Log inputs, prompt, model version, extracted fields, confidence, and any human override, enough to reconstruct any AI-influenced decision. Chain-of-thought reasoning kept for explainability.
  • NAIC model bulletin on "Use of AI Systems by Insurers" is adopted in 24 states and is effectively the national standard. Governance, testing, documentation, and bias controls belong in the design from day one.

Suggested first build · "we're not starting from zero"

A surety WIP + financials extractor: monitored mailbox → document classifier → Textract / multimodal read → structured extraction to a typed surety schema (principal, obligee, bond type/amount, per-project WIP rows, working capital) → validation + confidence → a bonding-capacity appetite check → a ranked triage queue, fully instrumented with confidence scores and an audit log. It maps one-to-one to how the market leaders describe their own systems, while the surety focus signals what a generic vendor cannot do.
